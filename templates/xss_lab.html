Why It Works

✓ Vulnerable version

User input is inserted directly into the HTML page.

Browser interprets it as JavaScript and executes it.

return f"<div>{comment}</div>"

✓ Secure version

User input is HTML escaped before rendering.

Browser displays it as text instead of executing it.

return render_template(
    "page.html",
    comment=escape(comment)
)

Defense

Jinja escapes values by default. Avoid |safe for untrusted data, validate inputs, and use a Content Security Policy for defense in depth.

OWASP recommendation

OWASP recommends contextual output encoding, safe templating, input validation, and Content Security Policy to reduce XSS impact.

Vulnerable vs secure implementation

The vulnerable version renders user input as HTML. The secure version escapes special characters so the browser displays text instead of executing script.
