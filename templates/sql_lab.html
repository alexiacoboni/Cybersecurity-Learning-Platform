SQL Injection

SQL injection happens when untrusted input is joined directly into a query. The defense is to bind data as parameters so the database treats it as values, not executable SQL.

Vulnerable examples are intentionally unsafe and exist only inside this lab.

Prepared Statement Pattern

db.session.execute(
    text("SELECT username FROM users WHERE username = :username"),
    {"username": user_input}
)

How to prevent it

Use parameterized queries, least-privilege database accounts, server-side validation, and avoid revealing SQL errors to users.

OWASP recommendation

OWASP recommends prepared statements with parameterized queries as the primary defense against injection.

Vulnerable vs secure implementation

The vulnerable version treats input as part of the SQL syntax. The secure version binds input as data, so the payload cannot change the query logic.
